Try Drone Guide

Everything you need to identify your task cell, load the correct flight file, and fly a pre-planned autonomous mission safely.

1Introduction

DroneTM coordinates drone mapping missions across teams of operators — dividing a large area of interest into manageable task cells, generating flight plans for each cell, and collecting the resulting imagery for processing into orthomosaics and point clouds.

This guide is for pilots who have downloaded a try-drone ZIP package containing pre-made flight plan tasks. You do not need a DroneTM account to fly these tasks — everything required is already in the folder.

2The Tasks

Task Grid Naming Convention

DroneTM divides the project area into a grid. Each cell is a self-contained task with its own flight plan. Task IDs encode a cell's position in the grid so you can immediately locate it on a map.

Rows run north to south. The northernmost strip is row A; the next strip south is row B; and so on. For grids with more than 26 rows, the prefix doubles: row 27 becomes AA, row 28 becomes AB, and so on.

Columns run west to east, numbered from 1. Column 1 is the westernmost column in each row. The combined ID is {row letter}{column number}.

ID Position
A1Top-left (northernmost row, westernmost column)
A2Top row, second from the left
B1Second row from the top, leftmost column
B3Second row, third from the left

When you download all tasks, the ZIP contains one file per cell named after its ID — for example A1.kmz or B3.plan. Match the file name to the cell label on your map before takeoff.

3Flying the Plan

Follow these steps for each task cell you intend to fly. Always complete the pre-flight checklist before takeoff — see the Docs section for the full 12-phase reference.

  1. 1
    Identify your drone model and the correct file

    Find your drone in the table below. Note the file extension in the Extension column. In the task ZIP, pick the file whose name matches your task ID and has that extension. A DJI Mini 4 Pro pilot flying task B2 would load B2.kmz.

  2. 2
    Transfer the file to your device

    Copy the flight file to the phone or tablet that runs your GCS app. For DJI models, you can also transfer via the DJI RC controller's internal storage. For QGroundControl, copy the .plan file to the device running QGC.

  3. 3
    Import the flight plan into your GCS app

    Open the GCS app listed for your drone model. Use its import or "open waypoints" function to load the file. Before proceeding, verify that the displayed flight path matches the task cell on your map — the lawnmower pattern should cover the correct area.

  4. 4
    Complete the pre-flight checklist

    Check airframe, propellers, and batteries, and confirm airspace clearance. Review imagery conditions. Do not skip this step. The DroneTM flight checklist covers 12 sequential phases from environment assessment through post-flight image inspection — link in the Docs section.

  5. 5
    Fly in autonomous mode

    Arm the drone and initiate the autonomous mission from your GCS app. Do not fly manually. The pre-planned route ensures consistent image overlap and altitude, which is required for the resulting imagery to stitch correctly into a usable orthomosaic. Stay within visual line of sight throughout.

  6. 6
    Inspect imagery and mark the task done

    After landing, review the captured images for blur, gaps, or exposure problems before leaving the site. Once satisfied, complete the task according to the package-specific instructions in the task card above.

FAQ

My drone model is not in the table. Can I still use these files?
The GeoJSON download works as a universal preview in any GIS tool, but it cannot be loaded directly to a drone. If your drone supports the QGroundControl .plan format (many ArduPilot and PX4-based drones do), try that option. For other models, check the DroneTM FAQ for current hardware support, or contact the package coordinator.
What altitude do these flight plans use, and can I change it?
The altitude was set when the flight plan was generated — typically between 70 m and 120 m above the takeoff point. It is baked into the file and cannot be changed without regenerating the plan. Higher altitude means faster coverage but lower ground resolution; lower altitude gives more detail but takes longer to fly. Use the DroneTM platform or the manual generation tool to produce a new file at a different altitude if needed.
Do I need a DroneTM account to use this package?
No. The flight files in this ZIP are self-contained and can be imported directly into your GCS app without logging into DroneTM. An account is only required if you want to claim tasks through the platform, upload imagery for automated processing, or create new projects. See the Drone Operators Guide for those workflows.
